## Root Cause Analysis
The `ApiKeyVerifier.verify_token` method in server.py was calling `self._verify(token)` without the `await` keyword. Since `_verify_bearer` is an async function, this returned a coroutine object instead of the tenant ID string. The coroutine object is truthy in Python, so it passed authentication checks but then failed when the code attempted to use it as a string, causing "Authentication failed" 400 errors on all MCP endpoint requests.

## Fix Implementation
- **File:** server.py
- **Line:** 75
- **Change:** Added `await` keyword before `self._verify(token)` call
- **Scope:** One-character fix
