### Trenton — Database & Data Pipeline Specialist

**Task:** Review Elliot's schema decisions before Darlene begins implementation.

**Artefact:** `.squad/agents/trenton/schema-review-v0.2.0.md`

**Critical catches (4):**

1. **Orphan-safe activity log** — `get_task_activity` should NOT require task existence; the activity log may contain entries for deleted tasks. Remove the task existence check to keep deleted-task audit trails accessible.

2. **FTS5 trigger validation** — Recommended adding a startup smoke-test (`SELECT fts5(1)` wrapped in try/except) to confirm FTS5 is compiled in before creating the virtual table. Prevents silent schema corruption on distros that ship SQLite without FTS5.

3. **Composite index recommendation** — `activity_log(task_id, created_at DESC)` composite index is more useful than the two separate single-column indexes Elliot specified. Single-column `task_id` index still helps for non-ordered lookups but the composite covers the primary query pattern.

4. **Blocking DNS in async SSRF check** — `socket.getaddrinfo()` is synchronous and blocks the event loop. Must be wrapped in `asyncio.get_event_loop().run_in_executor(None, ...)` for the webhook registration path.

### Mobley — Integration & External Systems Specialist

**Task:** Review Elliot's REST API and webhook design before Darlene begins implementation.

**Artefact:** `.squad/agents/mobley/api-review-v0.2.0.md`

**Critical catches (3):**

1. **`asyncio.create_task` GC bug** — `asyncio.create_task(_fire_webhooks(...))` returns a Task object that can be garbage-collected before completion if no reference is held. Fix: maintain a module-level (or closure-level) `_background_tasks: set` and call `_background_tasks.discard(task)` in a `task.add_done_callback`.

2. **`task.deleted` webhook fires after row deleted** — The delivery payload for `task.deleted` needs the compact task data, but if `_fire_webhooks` is called after the `DELETE` SQL executes, the row is gone. Fix: capture the task dict before deletion and pass it to `_fire_webhooks` explicitly.

3. **REST `update_task` logs before 404 check** — The activity log `_log()` call was placed before the existence check in the REST PATCH handler. A PATCH to a non-existent task ID would write a spurious activity log entry. Fix: check existence first, return 404 before logging.

### Dom — Security Expert

**Task:** Security audit of all v0.2.0 additions.

**Artefact:** `.squad/agents/dom/security-audit-v0.2.0.txt`

**Fixes applied (8):**

1. **IPv4-mapped IPv6 SSRF bypass (HIGH)** — `::ffff:192.168.1.1` format could bypass the RFC1918 check. Fixed by normalising resolved addresses through `ipaddress.ip_address()` and checking the `ipv4_mapped` attribute before blocklist evaluation.
2. **REST body size cap 1MiB (HIGH)** — REST endpoints had no body size limit; a large POST/PATCH could exhaust memory. Added 1MiB cap in REST auth/parsing middleware.
3. **`_FixArgumentsMiddleware` 6MB cap** — Existing middleware had no body size limit; added 6MB cap to prevent memory exhaustion via oversized MCP requests.
4. **FTS5 error message sanitized** — Raw SQLite FTS5 error messages were being returned to callers, potentially leaking internal schema details. Errors now return a generic "search failed" message.
5. **Tag length cap** — Individual tags had no length limit; added per-tag character cap.
6. **Tag count cap** — No limit on number of tags per task; added maximum tag count.
7. **`complete_tasks` ID validation** — Bulk complete was not validating ID format/length before querying; added same validation as single `complete_task`.
8. **REST required-field guards** — POST /tasks was not returning 400 on missing required fields in all code paths; guards hardened.

**Open item:** DNS rebinding — SSRF check is registration-time only. Low-TTL rebinding could pivot a registered HTTPS webhook to an internal address post-registration. Flagged for Elliot decision. See `decisions.md` OPEN ITEM 2026-04-01.
