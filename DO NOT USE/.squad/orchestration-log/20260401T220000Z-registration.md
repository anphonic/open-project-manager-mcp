## Agent Reports

### Elliot ‚Äî Lead & Architect

**Task:** Design self-service token registration feature.

**Deliverables:**
- `.squad/decisions.md` ‚Äî "Self-Service Token Registration" section (10 decisions)
- `.squad/agents/elliot/darlene-brief-register.md` ‚Äî full implementation brief for Darlene

**Key decisions:**

1. **Storage:** SQLite `tenant_keys` table (`squad TEXT PK`, `key TEXT`, `created_at TEXT`) ‚Äî idempotent `CREATE TABLE IF NOT EXISTS` in `_SCHEMA`
2. **Auth lookup:** Re-query DB on every auth check ‚Äî no cache, no restart needed on new registration
3. **Precedence:** Env var keys checked first (O(n) with constant-time compare), DB keys on miss
4. **`POST /api/v1/register`:** Returns `404` when `OPM_REGISTRATION_KEY` unset (feature disabled, zero surface area); `409` on duplicate squad; `201` + token on success
5. **Rate limiting:** In-memory per-IP counter ‚Äî `dict[str, list[float]]`, 5 attempts / 60s window, `429` on exceed, no external deps
6. **Token format:** `secrets.token_urlsafe(32)` ‚Äî consistent with `--generate-token` CLI
7. **Plaintext storage:** Consistent with existing local-first posture (env var tokens, webhook secrets all plaintext)
8. **`DELETE /api/v1/register/{squad}`:** Revocation via `X-Registration-Key` header; `204` success, `404` if not found
9. **`--generate-token` unchanged:** Stays stdout-only, no DB write
10. **`ApiKeyVerifier` refactored:** Accepts `verify_fn: Callable` ‚Äî shared `_verify_bearer` closure handles both env var and DB lookup
11. **MCP auth scope:** DB-registered keys grant REST access only. MCP auth still requires env var keys (`OPM_TENANT_KEYS`). `if tenant_keys:` guard on `token_verifier` / `auth_settings` preserved.
12. **`OPM_REGISTRATION_KEY` length:** Minimum 16 chars enforced at startup with `WARNING:` to stderr; no hard exit

### Darlene ‚Äî Backend Dev

**Task:** Implement self-service registration per Elliot's brief.

**Work completed in `server.py` and `__main__.py`:**

- **Schema:** `tenant_keys` table appended to `_SCHEMA` ‚Äî idempotent, no separate migration block
- **`_verify_bearer` closure:** Defined inside `create_server()` after `_lock`; env var keys first, DB re-queried on every call on miss; shared by both MCP verifier and REST `_check_auth`
- **`ApiKeyVerifier` refactored:** `__init__` now accepts `verify_fn: Callable` only; `verify_token` delegates entirely to the closure; class is now testable in isolation
- **`_check_auth` updated:** Unauthenticated mode (`no env keys + empty DB`) still returns `"system"` actor; authenticated path calls `_verify_bearer`
- **Rate limiter:** `_check_rate_limit(ip)` inside `_build_rest_router()` using `defaultdict(list)` + `time.monotonic()`; `_RATE_WINDOW=60.0`, `_RATE_MAX=5`
- **`_SQUAD_RE`:** `re.compile(r'^[a-zA-Z0-9_-]{1,64}$')` ‚Äî squad name validation
- **`register_endpoint`:** `POST /register`; reads `OPM_REGISTRATION_KEY` via `os.environ.get`; rate-limited; validates squad name; constant-time key compare; `409` on duplicate; inserts and commits; returns `201` + token + note
- **`deregister_endpoint`:** `DELETE /register/{squad}`; key in `X-Registration-Key` header; `204` on success via `Response(status_code=204)`; `404` if not found
- **Routes added:** `/register` (POST) and `/register/{squad:str}` (DELETE) in `_build_rest_router()` route list
- **`__main__.py`:** `OPM_REGISTRATION_KEY` length warning added after `_load_tenant_keys()` ‚Äî warns if set but shorter than 16 chars
- **New imports added to `server.py`:** `import os`, `import re`, `import secrets`, `import time`, `from collections import defaultdict`

**Tests:** 26 new tests in `tests/test_registration.py` ‚Üí **250 total** (all passing pre-Dom audit).
