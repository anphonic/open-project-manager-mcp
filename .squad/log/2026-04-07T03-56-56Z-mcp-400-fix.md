# Session Log: MCP POST /mcp Returns 400 Bad Request — Root Cause & Fix

**Date:** 2026-04-07T03:56:56Z  
**Topic:** Copilot CLI MCP 400 Bad Request Fix

## Summary

Fixed two compounding bugs that caused POST `/mcp` to return HTTP 400 instead of 401 for invalid/missing Bearer tokens:

1. **ApiKeyVerifier.verify_token() raised AuthenticationError** instead of returning None, violating the TokenVerifier protocol. Starlette's default_on_error converts exceptions to 400, not 401.

2. **_EarlyAuthRejectMiddleware rewrite incomplete** — still only guarded POST requests, never validated token value. Bad tokens reached FastMCP's auth layer where the 400 was generated.

## Root Cause 1: ApiKeyVerifier Protocol Violation

`ApiKeyVerifier.verify_token()` raised `AuthenticationError` on invalid tokens instead of returning `None`. This violates the `TokenVerifier` protocol contract. The exception propagated through Starlette's `AuthenticationMiddleware.default_on_error`, which returns HTTP 400 (not 401).

### Fix
```python
async def verify_token(self, token: str) -> AccessToken | None:
    try:
        tenant_id = await self._verify(token)
        if not tenant_id:
            return None  # protocol-compliant
        return AccessToken(token=token, client_id=tenant_id, scopes=["api"])
    except Exception:
        return None  # never raise
```

## Root Cause 2: _EarlyAuthRejectMiddleware Rewrite Incomplete

Middleware rewrite was partially applied:
- Signature still used `requires_auth: bool` (should be `tenant_keys: dict | None`)
- Only guarded POST requests with missing Authorization header
- Never validated token VALUE — `Authorization: Bearer wrongtoken` passed through
- Bad tokens reached FastMCP's auth layer where 400 was generated

### Fix
Complete rewrite of `_EarlyAuthRejectMiddleware`:
- Signature: `tenant_keys: dict | None` validates env-var keys only (no DB call)
- Guards ALL HTTP methods with `hmac.compare_digest` synchronous token validation
- Returns 401 with `Connection: close` and `more_body: False`
- Skips `/api/` paths (REST handles own auth)
- Runs BEFORE `SessionActivityMiddleware` (before session state or body buffering)

**Middleware order (outermost first):**
```
ConnectionTimeoutMiddleware
  _EarlyAuthRejectMiddleware     ← fast 401 + Connection: close before body read
    SessionActivityMiddleware
      _FixArgumentsMiddleware
        Starlette app
```
