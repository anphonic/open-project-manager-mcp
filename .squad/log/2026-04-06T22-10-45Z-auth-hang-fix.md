# Session Log: Auth-Hang Fix

**Date:** 2026-04-06  
**Topic:** Fix auth bypass + hang DoS on unauthenticated requests

---

## Summary

### Darlene ‚Äî Backend Implementation

**Bug Found:** When OPM receives HTTP requests to `/mcp` with missing or invalid Authorization headers, the server hangs and stops responding entirely.

**Root Cause (3 compounding issues):**
1. `_EarlyAuthRejectMiddleware` only guarded POST requests, not GET requests
2. Middleware only checked header *presence*, not token *value* ‚Äî `Authorization: Bearer wrongtoken` passed through
3. Missing `Connection: close` header left TCP connections in ambiguous keep-alive state

**Implementation:**
- Rewrote `_EarlyAuthRejectMiddleware` to:
  - Guard ALL HTTP methods (GET, POST, etc.)
  - Validate Bearer token value synchronously using `hmac.compare_digest` against env-var keys
  - Return proper 401 (not 400) with `Connection: close` and `more_body: False`
  - Skip `/api/` prefix (REST endpoints have their own auth)
  - Moved middleware BEFORE `SessionActivityMiddleware` to prevent bad-auth requests from touching session state or body buffer

**Middleware Stack (after fix, outermost first):**
```
ConnectionTimeoutMiddleware
  _EarlyAuthRejectMiddleware   ‚Üê fast 401 + Connection: close, before body read
    SessionActivityMiddleware
      _FixArgumentsMiddleware
        Starlette app (MCP + REST)
```

**Testing:**
- Added 7 regression tests in `tests/test_middleware.py::TestEarlyAuthRejectMiddleware`
- **394 tests passing** (non-telemetry tests)
- All new tests covering: no tenant keys, POST/GET without auth, wrong token, valid token, /api/ bypass, lifespan scope

### Dom ‚Äî Security Review

**Primary Fix Verification:**
- Confirmed missing `await` on `_verify_bearer` was root cause of accepting all Bearer tokens
- Darlene's `await` fix is correct and closes the primary attack vector

**Additional Fixes Applied:**
1. `_EarlyAuthRejectMiddleware` placed BEFORE `_FixArgumentsMiddleware` (body buffering)
   - Prevents 6 MB body reads on unauthenticated POST requests
   - Mitigates Slowloris-style DoS via body amplification
   - With 100 concurrent requests: 600 MB memory saved per rejected batch

2. Auth error messages sanitized
   - `"Invalid API key"` ‚Üí `"Unauthorized"` (no mechanism leakage)
   - `"Authentication failed"` ‚Üí `"Unauthorized"`
   - Prevents auth mechanism type disclosure via Starlette error handler

**Security Analysis Results:**
- ‚úÖ Unauthenticated requests cannot reach MCP transport (three-layer defense)
- ‚úÖ No state accumulation from failed auth attempts
- ‚úÖ Malformed requests cannot wedge event loop (multiple safeguards)
- ‚úÖ Auth errors don't leak token values or internal state
- ‚úÖ Connection lifecycle properly managed (Connection: close, timeouts, rate limits)

**Testing:**

- All 13 middleware tests passing
- No regressions from security fixes
