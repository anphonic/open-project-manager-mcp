### Dom — Security Expert *(newly onboarded this session)*

**Task:** Security audit of `server.py`, `__main__.py`, and transport layer.

**Findings and fixes:**
1. `update_task` column names were interpolated via f-string — currently unexploitable (columns are hardcoded in caller), but not fail-safe → added `_VALID_UPDATE_COLUMNS` allowlist; raises `ValueError` on unknown column
2. `list_tasks` and `list_ready_tasks` accepted arbitrary `limit` values — potential DoS via large result sets → added `_MAX_LIMIT = 500` cap
3. `conn.execute()` calls were not wrapped — SQLite errors would propagate raw stack traces → wrapped all in `try/except sqlite3.Error`
4. No input length validation on text fields — oversized inputs could cause unbounded writes → added `_MAX_SHORT_FIELD = 500` for id/title/project/assignee and `_MAX_DESCRIPTION = 50_000` for description; enforced in `create_task` and `update_task`
5. Path traversal via `--db-path` CLI flag — documented as Info/low risk for v1 single-tenant; no fix required at this time
