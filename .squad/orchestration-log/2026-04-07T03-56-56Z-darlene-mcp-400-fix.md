# Orchestration Log: MCP POST /mcp Returns 400 Bad Request — Root Cause & Fix

**Date:** 2026-04-07T03:56:56Z  
**Agent:** Darlene (Backend Implementation)  
**Incident:** POST /mcp returning HTTP 400 instead of 401 for invalid/missing Bearer tokens

---

## Incident Summary

Copilot CLI MCP client (type: "http") received **HTTP 400 Bad Request** on every POST to `/mcp`, even though REST `/api/v1/stats` returned 200 with the same token. This rendered the MCP endpoint unusable.

---

## Root Cause Analysis

### Root Cause 1: ApiKeyVerifier Raises on Auth Failure

**File:** `src/open_project_manager_mcp/server.py`

**Issue:** `ApiKeyVerifier.verify_token()` raised `AuthenticationError` on invalid tokens instead of returning `None`.

**Protocol violation:** The `TokenVerifier` interface (from `mcp.server.auth.provider`) specifies:
```python
async def verify_token(self, token: str) -> AccessToken | None:
    # Must return None for rejected tokens, never raise
```

**Consequence:** Exception propagated through:
```
FastMCP BearerAuthBackend.authenticate()
  → raises AuthenticationError
    → Starlette AuthenticationMiddleware.dispatch()
      → default_on_error handler
        → PlainTextResponse(status_code=400)  ← WRONG: should be 401
```

### Root Cause 2: _EarlyAuthRejectMiddleware Rewrite Incomplete

**File:** `src/open_project_manager_mcp/__main__.py`

**Issue:** Middleware rewrite partially applied:
1. Signature still used `requires_auth: bool` instead of `tenant_keys: dict | None`
2. Only guarded POST requests with **missing** Authorization header
3. Never validated token **VALUE** — `Authorization: Bearer wrongtoken` passed through
4. Bad tokens reached FastMCP's auth layer where the 400 was generated

**Why REST worked:** REST endpoints call `_check_auth()` directly, which returns JSONResponse 401 on failure, bypassing Starlette's `AuthenticationMiddleware` entirely.

---

## Solution

### Fix 1: ApiKeyVerifier.verify_token() Returns None

**Before:**
```python
async def verify_token(self, token: str) -> AccessToken | None:
    tenant_id = await self._verify(token)
    if not tenant_id:
        raise AuthenticationError("Invalid API key")  # ❌ WRONG
    return AccessToken(...)
```

**After:**
```python
async def verify_token(self, token: str) -> AccessToken | None:
    try:
        tenant_id = await self._verify(token)
        if not tenant_id:
            return None  # ✅ protocol-compliant
        return AccessToken(token=token, client_id=tenant_id, scopes=["api"])
    except Exception:
        return None  # ✅ never raise
```

**Result:** `BearerAuthBackend` gets `None`, returns `None` itself, Starlette sets `UnauthenticatedUser`, `RequireAuthMiddleware` sends proper 401.

### Fix 2: _EarlyAuthRejectMiddleware Complete Rewrite

**Before:**
```python
async def _EarlyAuthRejectMiddleware(app, scope, receive, send):
    if scope["type"] != "http":
        await app(scope, receive, send)
        return
    
    # Only checked POST + missing header
    if scope["method"] == "POST":
        if not scope["headers"].get(b"authorization"):
            # sent 401 but no Connection: close
```

**After:**
```python
async def _EarlyAuthRejectMiddleware(app, scope, receive, send):
    # Guards ALL methods (GET, POST, etc.)
    # Validates token VALUE via hmac.compare_digest
    # Returns 401 with Connection: close before body read
    # Skips /api/ (REST has own auth)
```

**Key improvements:**
- **Method coverage:** Guards all HTTP methods (GET, POST, PUT, DELETE)
- **Token validation:** Validates Bearer token value synchronously using `hmac.compare_digest` (constant-time) against env-var keys only
- **Early rejection:** Returns 401 with `Connection: close` and `more_body: False` before `SessionActivityMiddleware` or body buffering
- **REST exclusion:** Skips `/api/` prefix (REST endpoints handle own auth)

**Middleware order (outermost first):**
```
ConnectionTimeoutMiddleware
  _EarlyAuthRejectMiddleware     ← rejects invalid tokens before state mutation
    SessionActivityMiddleware
      _FixArgumentsMiddleware
        Starlette app (MCP + REST)
```

## Key Learnings

1. **TokenVerifier protocol** — Any implementation that raises exceptions will cause Starlette's `default_on_error` to return 400. Must return `None` for auth failures.

2. **Middleware ordering matters** — Auth rejection should happen before session state accumulation to prevent resource exhaustion from unauthenticated requests.

3. **Connection lifecycle** — Include `Connection: close` on rejection responses to prevent keep-alive ambiguity that can wedge TCP connections.

4. **Defense in depth** — Double-layer token validation (env-var check + FastMCP DB check) catches both simple typos and compromised tokens early.
